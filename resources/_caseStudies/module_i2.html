Module I

Week 11

Monday, 18th April, 2005 – Sunday, 24th April, 2005

Story:

On the Monday of the last week of the semester, Denise tries to set up a time for a final code review meeting:

Date: Mon, 18 Apr 2005 12:58:55 -0400 (EDT)

Subject: [Fwd: Re: Meetings]

From: Denise

To: crane-cs-l@mtu.edu, Hank

 

 

The CS Senior Design Crane Team, Dr. Voelker, and Dr. Taylor, need to have a code review/final meeting sometime this week. Below are the times that Dr. Voelker cannot make it, and I am unavailable Tuesday/Thursday 8- 11:30 and 3:30-5, Wednesday 1-5.

What does everyone else's schedule look like? Is it possible to meet Friday morning?

--Denise

 

Bob wonders about the meeting:

 

Date: Wed, 20 Apr 2005 11:41:14 -0400

From: Bob

To: Crane Team <crane-cs-l@mtu.edu>

Subject: Meeting Tomorrow?

 

Just curious as to if we are meeting tomorrow before class? I will have a beta final report out by the end of today.

-Bob

 

Then sends out a draft of the final report, seeking comments:

 

Date: Wed, 20 Apr 2005 20:54:51 -0400

From: Bob

To: Crane Team <crane-cs-l@mtu.edu>

Subject: Skeleton Doc.

 

Attached is a skeleton(or alpha) version of the Final Report.

I want to discuss some of the sections with you tomorrow at class.

There is certain parts that need to be updated or need the latest version in the doc, such as the risk analysis document.

I also want to make sure the layout is fine with you guys.

After we meet I will update the document and send out a Beta version tomorrow night.

-Bob

 

Attachment Converted: "Crane 2\Attach\final project.pdf"

 

 

On Thursday, Denise announces the time of the Final Code review meeting:

 

Date: Thu, 21 Apr 2005 14:38:49 -0400 (EDT)

Subject: Crane Code Team Final Meeting

From: Denise

To: crane-cs-l@mtu.edu, Hank

 

Dr. Voelker, Dr. Taylor, CS4790 students

 

Our final meeting / code review for the SeaBase Crane Code Senior Design

Project will be tomorrow 4/22/05 at 8am in Rehki CS Hall 217 (conference room).

 

Hank, would you be able to bring one of the laptops from your lab with MatLab installed?

 

Thank you,

--Denise

 

And Justin submits the updated model, with reservations:

 

Date: Thu, 21 Apr 2005 15:28:33 -0400

From: Justin

 

here's the updated model, there is subsystems of subsystems :) fun stuff, anyway i had to rename setup-run.c to setup_run.c, because i couldn't get it to compile otherwise.  i'm not sure if i have all the inputs right...i have multiple papers with different numbers on it.  also, be sure to change the setup_run.c file to reflect the change (up at the top).

 

i couldn't get it to compile, so there is no parameters specified, make sure to change the model so it can read params.

 

any questions let me know,

-Justin
